# Codebase Structure

## Root Directory
```
shotgun-cli/
├── main.go                      # Application entry point with platform-specific init
├── package.json                 # npm configuration and build scripts
├── go.mod                       # Go module definition and dependencies
├── .gitignore                   # Git ignore patterns
├── CLAUDE.md                    # Claude Code documentation
└── README.md                    # Project documentation
```

## Key Directories

### `/internal/` - Internal Application Code
- **`/internal/core/`**: Business logic and core functionality
- **`/internal/ui/`**: BubbleTea UI components and views

### `/templates/` - Built-in Prompt Templates
- `prompt_makeDiffGitFormat.md` - Dev template (git diff format)
- `prompt_makePlan.md` - Architect template (planning document)
- `prompt_analyzeBug.md` - Debug template (bug analysis)
- `prompt_projectManager.md` - Project template (documentation sync)

### `/bin/` - Build Outputs
- Cross-platform binaries generated by build process
- `shotgun-cli-wrapper.js` - npm wrapper script

### `/scripts/` - Build and Installation Scripts
- Node.js build scripts for cross-platform compilation
- Installation scripts for npm package

## Core Package Details

### `internal/core/` - Business Logic
- **`types.go`**: Core data structures (FileNode, DirectoryScanner, etc.)
- **`scanner.go`**: Directory scanning with gitignore integration
- **`generator.go`**: Context generation and prompt creation
- **`template*.go`**: Template system (built-in and custom templates)
- **`config*.go`**: Configuration management and validation
- **`enhanced_*.go`**: Advanced features (translation, circuit breaker, retry logic)
- **`*_test.go`**: Comprehensive test coverage

### `internal/ui/` - User Interface
- **`app.go`**: Main BubbleTea model with ViewState management
- **`views.go`**: View rendering for each application step
- **`filetree.go`**: Interactive file tree with exclusion toggling
- **`components.go`**: Reusable UI components and styling
- **`config_*.go`**: Configuration UI components
- **`*_test.go`**: UI component tests

## Application Flow
1. **FileExclusion**: Interactive file tree for excluding files
2. **TemplateSelection**: Choose from built-in or custom templates
3. **TaskDescription**: User describes their task
4. **CustomRules**: Optional rules and constraints
5. **Generation**: Create and save the prompt

## Configuration Locations
- **Windows**: `%LOCALAPPDATA%\shotgun-cli\`
- **macOS**: `~/Library/Application Support/shotgun-cli/`
- **Linux**: `~/.config/shotgun-cli/`

## Template System Architecture
- **Built-in templates**: Embedded in binary from `/templates/` directory
- **Custom templates**: User-defined in platform-specific config directory
- **Template variables**: `{TASK}`, `{RULES}`, `{FILE_STRUCTURE}`, `{CURRENT_DATE}`
- **YAML frontmatter**: Metadata for custom templates
